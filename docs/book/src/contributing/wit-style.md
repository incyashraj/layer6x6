# WIT style guide

> This guide is written in Phase 2 when the first WIT files are authored.

It will cover:

- Package naming (`layer36:<module>@<version>`)
- Interface naming (kebab-case)
- Function naming (kebab-case, imperative verbs)
- Resource lifetimes
- Error types and variants
- When to use `result<T, error>` vs. trapping
- Comment conventions (WIT comments appear in the generated reference docs)
- The "native three of five" test for widget types

Until Phase 2 ships, refer to the
[WIT specification](https://component-model.bytecodealliance.org/design/wit.html)
and the Bytecode Alliance's own WIT conventions.
